README 📖

I'm a builder — 13 years of it in telecom. The domain taught me the discipline; the method transfers to any industry. Every system on this page began the same way: a problem nobody assigned me, stripped to first principles, automated, and shipped. Telecom-grown, industry-agnostic — the next problem doesn't need to have an antenna on it.

First principles

Strip a problem to what's actually true — the data, the physics, the user's pain — and build up from there, not from how it's always been done.

Systems thinking

No fix in isolation: every tool considers the pipeline upstream, the engineer downstream, and the feedback loop that keeps it honest. Optimize the system, not the symptom.

Growth mindset

Field → RF → performance → data → automation → AI. Every stage self-taught before it was job-required. The next skill is always in progress.

Automation

If a human does it twice, build the pipeline. 12 production tools, 116 AI skills, and 60%+ of manual analysis eliminated by following that rule.

Value creation

The only metric that counts: hours returned to engineers, decisions made faster, problems that stay solved. Tools are a means; value is the product.
